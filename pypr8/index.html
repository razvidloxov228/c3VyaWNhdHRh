1.Основи HTTP. Використання у веб-розробці. Основні методи

HTTP

HTTP(Hypertext Transfer Protocol) це протокол обміну даними між клієнтом і сервером. Він використовується для запитів і отримання ресурсів у веб-розробці.

Основні методи

  • GET: Використовується для запиту ресурсу. Не змінює стан сервера. Запити можуть кешуватися та залишаються в історії браузера.
  • POST: Використовується для створення нового ресурсу або виконання дії. Може змінювати стан сервера. Дані передаються в тілі запиту.
  • PUT: Замінює існуючий ресурс новим. Є ідемпотентним, повторний запит дає однаковий результат.
  • DELETE: Використовується для видалення даних із сервера. Також є ідемпотентним.

2.Клієнт та Сервер. Різниця. Взаємодія

Клієнт

  • Це будь-який пристрій або програмне забезпечення, наприклад браузер чи мобільний додаток, що здійснює запит до сервера.
  • Основна роль клієнта це надсилати запити та відображати отримані дані користувачеві.
  • Приклад: браузер Chrome, що відкриває веб-сторінку.

Сервер

  • Це комп’ютер або програма, що зберігає, обробляє та надає дані клієнтам.
  • Сервер може виконувати бізнес-логіку, працювати з базами даних, перевіряти права доступу.
  • Приклад: веб-сервер Apache або Nginx, який віддає HTML-сторінку чи API-відповідь.

Взаємодія сервера та клієнта.

HTTP працює за принципом запит–відповідь:

  1. Клієнт формує HTTP-запит
    • Вказує метод (GET, POST, PUT, DELETE), URL-адресу ресурсу, заголовки (headers) та, за потреби, тіло запиту (body).
    • Наприклад: GET /index.html HTTP/1.1
  2. Сервер отримує запит і обробляє його
    • Виконує потрібну дію: читає файл, звертається до бази даних, запускає бізнес-логіку.
    • Формує відповідь із кодом статусу, наприклад: 200 OK, 404 Not Found та даними.
  3. Клієнт отримує HTTP-відповідь
    • Відповідь містить статус, заголовки та тіло, наприклад: HTML, JSON, зображення.
    • Браузер відображає сторінку, або додаток обробляє дані API.
